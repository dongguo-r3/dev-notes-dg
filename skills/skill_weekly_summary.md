# Skill: Weekly Summary

## Trigger
Run this at the end of the week, or when the user asks for a weekly summary or weekly digest.

## Steps
1. Open `MASTER.md` and read all daily log entries from the past 7 days.
2. Open each active worker file and note current Status and any open Blockers.
3. Compile the summary using the output format below.
4. Present the summary to the user.
5. Ask if the user wants to append it to MASTER.md as a `## Weekly Summary` section.

## Output Format

## Weekly Summary — Week of YYYY-MM-DD

### Progress This Week
- **[project-name]**: [2–3 sentence summary of the week's progress]
- **[project-name]**: [2–3 sentence summary]

### Completed This Week
- [project-name] — archived on YYYY-MM-DD

### Still Blocked
| Project | Blocker | Duration |
|---|---|---|
| project-alpha | Waiting on design review | 3 days |

### Looking Ahead
_(Optional: note any projects expecting significant activity next week)_
